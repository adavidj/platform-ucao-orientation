Contexte

L'UCAO (Université Catholique de l'Afrique de l'Ouest) souhaite se doter d'une plateforme numérique dédiée à l'orientation des étudiants — notamment les nouveaux bacheliers, les étudiants en réorientation et toute personne désirant s'inscrire pour la première fois à l'UCAO. Cette plateforme viendra digitaliser et personnaliser l'expérience d'orientation qui se fait actuellement de manière manuelle et peu structurée.

Objectifs principaux
  • Présenter l'UCAO : ses filières, écoles, facultés et formations.
  • Orienter les visiteurs : via un formulaire intelligent qui propose des filières adaptées au métier souhaité.
  • Générer un rapport d'orientation personnalisé : envoyé par e-mail et téléchargeable en PDF.
  • Collecter les préinscriptions : avec envoi automatique d'un e-mail de confirmation de réception.
  • Administrer la plateforme : via un tableau de bord sécurisé réservé aux administrateurs.
  • Améliorer la communication : en utilisant les données collectées pour cibler les futurs étudiants.

Tableau de Bord Administrateur

Accès & Authentification
  • URL d'accès : /admin/
  • Authentification sécurisée (login + mot de passe hashé bcrypt)
  • Protection CSRF sur tous les formulaires admin
  • Gestion des sessions avec expiration automatique
  • Niveaux d'accès : Super Admin / Admin standard
Module — Dashboard Principal
  • Nombre d'orientations effectuées (total / ce mois)
  • Nombre de préinscriptions reçues (total / ce mois)
  • Nombre de rapports PDF générés et envoyés
  • Métiers les plus demandés (graphique en barres — Chart.js)
  • Filières les plus recommandées (graphique en camembert — Chart.js)
  • Dernières orientations et préinscriptions (tableau récapitulatif)
Module — Gestion des Orientations
  • Liste paginée : Nom/Prénom, E-mail, Métier souhaité, Filières recommandées, Date, Statut envoi e-mail
  • Bouton : Télécharger le rapport PDF généré pour chaque orientation
  • Bouton : Envoyer une réponse personnalisée (pop-up)
  • Filtres : par date, série bac, métier, filière recommandée
  • Export CSV / Excel
Pop-up "Réponse personnalisée" : Message pré-rempli automatiquement selon le métier choisi par le visiteur, modifiable par l'admin avant envoi direct par e-mail depuis l'interface.
Module — Gestion des Préinscriptions
  • Liste complète paginée de tous les préinscrits avec l'ensemble de leurs informations
  • Bouton : Voir le dossier complet d'un préinscrit
  • Bouton : Télécharger les documents soumis par le préinscrit
  • Bouton : Envoyer un e-mail de notification au préinscrit
  • Bouton : Exporter la liste complète en PDF
  • Export CSV / Excel
  • Filtres : par filière, par date, par série bac
Module — Notifications
  • Composer et envoyer des e-mails groupés : à tous les orientés, à tous les préinscrits
  • Ciblage par filière ou par série bac
  • Historique complet des notifications envoyées (sujet, message, destinataires, date)
Module — Filières & Métiers
  • Gestion de la table de correspondance Métiers ↔ Filières (CRUD complet)
  • Gestion des filières : ajout, modification, description, débouchés
  • Activation / désactivation de la période nouveaux bacheliers (active le champ "Numéro de table" dans le formulaire d'orientation)

Base de Données — Schéma

Table : orientations
-- Toutes les orientations effectuées via le formulaire
TABLE orientations (
  id                    INT AUTO_INCREMENT PRIMARY KEY,
  nom                   VARCHAR(100),
  prenom                VARCHAR(100),
  email                 VARCHAR(150),
  telephone             VARCHAR(20),
  serie_bac             VARCHAR(10),
  numero_table          VARCHAR(30) NULL,
  metier_souhaite       VARCHAR(200),
  filieres_recommandees TEXT,
  rapport_pdf_path      VARCHAR(255),
  email_envoye          TINYINT(1) DEFAULT 0,
  created_at            DATETIME DEFAULT CURRENT_TIMESTAMP
)
Table : preinscriptions
-- Préinscriptions reçues — confirmation par e-mail uniquement
TABLE preinscriptions (
  id                    INT AUTO_INCREMENT PRIMARY KEY,
  nom                   VARCHAR(100),
  prenom                VARCHAR(100),
  date_naissance        DATE,
  nationalite           VARCHAR(80),
  serie_bac             VARCHAR(10),
  annee_bac             YEAR,
  etablissement_origine VARCHAR(200),
  filiere_choisie       INT -- FK → filieres.id,
  niveau_entree         VARCHAR(20),
  email                 VARCHAR(150),
  telephone             VARCHAR(20),
  doc_releve            VARCHAR(255) NULL,
  doc_bac               VARCHAR(255) NULL,
  doc_identite          VARCHAR(255) NULL,
  orientation_id        INT NULL -- FK → orientations.id (si venu via orientation),
  created_at            DATETIME DEFAULT CURRENT_TIMESTAMP
)
Tables : filieres, metiers, metiers_filieres
TABLE filieres (
  id            INT AUTO_INCREMENT PRIMARY KEY,
  ecole_faculte VARCHAR(100),
  nom_filiere   VARCHAR(200),
  niveau        VARCHAR(50),
  duree         VARCHAR(20),
  description   TEXT,
  debouches     TEXT,
  slug          VARCHAR(200)
)

TABLE metiers (
  id         INT AUTO_INCREMENT PRIMARY KEY,
  nom_metier VARCHAR(200),
  slug       VARCHAR(200)
)

TABLE metiers_filieres (
  id         INT AUTO_INCREMENT PRIMARY KEY,
  metier_id  INT -- FK → metiers.id,
  filiere_id INT -- FK → filieres.id,
  priorite   INT DEFAULT 1
)
Tables : notifications, admins, parametres
TABLE notifications (
  id            INT AUTO_INCREMENT PRIMARY KEY,
  sujet         VARCHAR(255),
  message       TEXT,
  destinataires TEXT,
  type_cible    VARCHAR(50),
  envoye_le     DATETIME,
  admin_id      INT -- FK → admins.id
)

TABLE admins (
  id            INT AUTO_INCREMENT PRIMARY KEY,
  nom           VARCHAR(100),
  email         VARCHAR(150) UNIQUE,
  password_hash VARCHAR(255),
  role          ENUM('super_admin','admin'),
  actif         TINYINT(1) DEFAULT 1,
  created_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
  last_login    DATETIME NULL
)

TABLE parametres (
  cle         VARCHAR(100) PRIMARY KEY,
  valeur      TEXT,
  description VARCHAR(255)
)
-- Exemple : periode_nouveaux_bacheliers = '1' active le champ numéro de table

Contraintes & Exigences Non Fonctionnelles

  • Responsive design : Compatible mobile, tablette et desktop (Bootstrap 5)
  • Performance : Temps de chargement cible < 3 secondes
  • Accessibilité : Respect des normes WCAG 2.1 niveau AA
  • Compatibilité navigateurs : Chrome, Firefox, Edge, Safari (dernières versions)
  • Sécurité RGPD : Mention de collecte de données, consentement, suppression sur demande
  • Sauvegarde : Backup automatique de la base de données recommandé (cron quotidien)
  • Langue : Français uniquement (version 1.0)
  • HTTPS : Certificat SSL obligatoire sur le serveur de production
  • Uploads sécurisés : Types de fichiers restreints (PDF, JPG, PNG), taille max 5 Mo par fichier
